                  Robert Scholte added a comment - 07/Sep/11 4:45 PM

                  During the generation of the Parse we need to specify the stacksize, which is 500 by default. 

                  -Jstack=<NNN> Changes the stack size from default 500 to NNN.

                  (from http://byaccj.sourceforge.net/)
                  In very, very rare cases this is not enough, and it seems you have hit such case (it's a complex class).
                  The problem is, that is can't be set afterwards:

                  final static int YYSTACKSIZE = 500;  //maximum stack size

                  I don't feel like changing this value, because what should it be? 600, 700, 2500?
                  The best solution is probably to build QDox yourself and change this value. (One person listed this issue before and came with this solution all by himself)
                  What I can do is document it and provide a maven-property to set it.
